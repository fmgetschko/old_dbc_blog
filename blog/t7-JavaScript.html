JS vs Ruby

So alike and sooo diferent...!

April 12, 2015

So, after one week of JavaScript, we're supposed to talk about one difference between JS and Ruby. Although they're both object-oriented languages, they are very different when you put them into practice, and that means, code. I chose to talk about looping in Ruby and JS.

Both in Ruby and JS, there are many different ways to write a loop. In general, both logic and operators to write loops and conditions are pretty similar. Still, you don't have to dig much to find the differences in their syntaxes.

I'll approach first the while loop. For that, I'll use an example of EJS book. A loop in JS would look like this:

while (counter < 10) {
result *= 2;
counter ++;
}

It is possible to write this code in a very similar way in Ruby. It would be like this:

while counter < 10 do
result *= 2
counter +=1
end

That works, but, in fact, there's a leaner way to do the same in Ruby. It would be like this:

10.times {result *= 2}

And this is the magic of Ruby! Ruby is designed to be very lean, and there's always a way to do what you want very easily (and shortly).

Well, the same applies to for loops. EJS example:

for (var number = 0; number<=12; number+=2)
console.log(number);

Using for loop in Ruby, it would be like this:

for i in 0..6 {puts i*2}

And there's also the loop that iterates over an array. In JS, it would look like this:

for (x in person){
text += person[x];
}

This type of loop is made with #each in Ruby, and looks like this:

person.each {|name| text += name}

In short, the loop logics in Ruby and JS are very similar. The biggest difference is that usually, Ruby syntax is simpler and leaner. The good news is that, once you're familiar with any of the logics, you won't have dificculties in applying the loops in the other language!
