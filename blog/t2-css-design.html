IDs vs Classes

Do you want to get specific or general?

March 8th, 2015

I’m just starting to explore this magical world of CSS. It’s impressive how much can be done by it and how lean and effective style got after CSS. Still, sometimes CSS can be confusing (at least for beginners, like me!). One good example of it refers to the use of IDs and Classes.

At first, Classes and IDs may seem disturbingly similar, if not the same thing. After all, by adding a class or an id to a tag, it is possible to define specific styles to the class/ id on your CSS stylesheet, and it will be automatically applied to that tag. So, if both accomplish to style your tag with the elements you wish, what’s the difference between them?

The difference is about specificity. Classes are designed to be used in several elements, as a set of style rules that you may want to apply in multiple tags. IDs, instead, are unique. They are designed to reference one single element, and shouldn’t be repeated in any other tags in your HTML. They can even be used together, in the same tag, if you wish to!.

Also, IDs have an special feature that you can’t find in Classes. By using IDs, you can set a scroll browser functionality, that is pretty cool. Imagine that you have a website http://mywebsite.com. If you have set up a tag with an ID, for example 'div id=“unique_id”', you can scroll to this element by using the URL http://mywebsite.com#unique_id. Isn’t it awesome?

In short, to guarantee you’ll never confuse it again, keep in mind: IDs are unique, Classes are general. Just memorize that, and you’ll hit it overtime!
