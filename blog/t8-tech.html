Active Record:

Using Ruby syntax to manage your database

April 15, 2015

Among several options that we were given this week, I chose to talk about a very interesting subject that is Active Record. Before I start talking about it, it is essential to talk about something else: Object-Relational Mapping, or ORM. So, what is ORM? ORM is a technique that connects tables in a relational database to objects in an application. By making this connection, ORM allows developers to use these object's properties and relationships to manage the database tables. And that means managing databases without having to write SQL statements!

And what about Active Record? Well, Active Record is an ORM framework. More specifically, the ORM framework found in Rails. So, if you build your application in Rails, you can use Active Record to represent your DB tables as models, that is, instances (or objects) of Ruby classes.

Let me exemplify that. Imagine that you have a table of international direct dialings of several countries. This table's name is idds and it has two fields: country_name and idd_code. You want to insert a new record in this table. Normally, you would use SQL to do that, and it would look like this:

INSERT INTO idds (country_name, idd_code) VALUES ('Brazil', 55);

But, if you're using Active Record, you can make the same thing by creating an instance of a class Idd, using Ruby! Like this:

idd = Idd.new
idd.country_name = "Brazil"
idd.idd_code = 55
idd.save

And, not only you can do that, but Active Record also allows you to validate the data to be inserted - like validation of non-empty values, or duplicated information - before it gets written (or persisted) to the database.

That looks awesome, doesn't it? So, from now on, create, read, update and delete information without writing a single SQL statement, and be happy! :)
