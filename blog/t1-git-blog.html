How version control, git and GitHub can make your life easier

Feb 26, 2015

As a business consultant, it wasn’t rare to spend nights up working on spreadsheets and powerpoint presentations. And, not rarely too, after spending hours working in a huge analysis…. bang! Excel just stopped working! Invariantly (even though Excel has an autosave feature), hours of work were waisted and I had to start all over again (and remembering how to structure your analysis again from scratch at 2am is not always that easy!). But what if, for each step of the analysis I evolved, I could just save a snapshot of it, controlling what’s been achieved so far? And, if something went wrong, or I didn’t like the results of my current iteration, I could then just roll back to my previous state, effortlessly?

The version control tool git does it, and more! Working with a tree structure, git defines a ‘master’ version of your document, from which you can derive as many branches as you need to keep working. Each branch starts as an identical copy of your master, and you’re free to make any changes you want to it without affecting master. Once you’ve come to a point worth ‘saving’, you can merge your changes back to the master, specifying what’s been made. All versions of this master document are preserved, and if you need to come back to any of them, you can simply do that by hitting one command line.

And if you have a team working in this same document, it gets even better. For collaborative work, git works beautifully along with GitHub. GitHub is a remote repository to which all your team can have access. The master will always be the most up to date version of the team’s work. Each person of the team will work on their part from their own (local) branch, and once they’re finished, they’ll merge their improvements to team’s (remote) master. The newest version of the document is already available to all the team to sync and keep working!

Moreover, if you want to maintain your project the highest level, there’s another amazing feature that GitHub provides: instead of directly merging your improvements to the project, you can instead ask for someone in the group to revise it for you. This is really useful not only as a double check, but also to keep the group always aware of how project’s growing and evolving overall.

If you’re interested in trying this miracle out, I would suggest you to take a look at this quick guide GitHub provides to set up git and GitHub and enjoy this beauty. Good luck!
